## What is GTM? * GTM is a conduit for scripts and tags you wish to put on your website. * It is a powerful and (reasonably) flexible tool that allows for built-in and custom scripts to be added to the website with the intention of little to no input from the developer. * In reality there is more developer input than people generally account for. Actually a lot more.
## The interface The main features of the GTM interface include:- * The container * Tags * Triggers * Variables
### Container * Has all of the tags that will appear on the a site. * Represented by a piece of code that goes on every page on your site.
### Tags * Script that you want to appear on the website * Some tags are quite easy to implement in the tag manager interface e.g. "on this page load this script". * Some are more difficult e.g. "on the confirmation page make the transaction information available to be picked up by tag manager".
### Triggers * The rule for why the tag appears on the website e.g. All pages, particular event, particular page * GTM provides a series of built in events that can be used in triggers e.g. clicks, form, timers, or you can create your own which can then be picked up by the trigger.
### Variables * Information that can be used in the trigger rule or the tag setup * Variables can be set as constants in the interface or pulled from a variety of places; dataLater is most common, but you would grab it from a dom element on the page (for example)
### Other stuff * Folders - Used for filing linked tags, triggers and variables because the list can get a bit unwieldy. * Version control for published containers * Preview and debug mode for unpublished (and published) containers
## The Data Layer * The more difficult tags are always where the website needs to interact with GTM. * The container will either pick up data when it is initialised e.g. GA setup, or react to data that is pushed after initialisation e.g. virtual pageview * SHOW A PICTURE OF THIS before, init, after * Communication is provided via the dataLayer array which data is pushed into. * The dataLayer can hold info for use in variables (macros) or events that need to be fired.
### Example items put in the dataLayer * Google UA Id. This would be put on the page before GTM initialisation * Site info e.g. logged in, 
						
 /* general check to see if the dataLayer exists already */
 var dataLayer = window.dataLayer || [];

 dataLayer.push({
	googleAnalyticsUA: '@Model.GoogleAnalyticsUA'
 });
